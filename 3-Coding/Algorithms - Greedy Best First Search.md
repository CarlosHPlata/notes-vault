# ðŸ“ƒ Greedy Best-First Search 23-06-2025

---
**Greedy Best-First Search** is a search algorithm to traverse a [[Data Structures - Graphs|Graph]] that expands the node that appears to be closest to the goal, based on a [[Algorithms - Heuristics in search algorithms|heuristic function]] $d=h(N)$. It uses a data structure that prioritizes nodes according to their heuristic value $h(N)$, always selecting the most "*promising*" node next. Since the decision depends on comparing heuristic values, the structure must maintain a **sorted order** after each insertion. A common choice for this is a [[Data structures - Priority Queue|min-priority queue]]

> [!tip] Search Algorithms
> Greedy Best-First Search is part of algorithms called **Search Algorithms** designed to traverse and search values in [[Data Structures - Graphs|Graphs]] more specific is part of the so called [[Ai - Informed vs Uninformed Search problems|Informed Search Algorithms]].

## Core Idea
- Create a **priority queue** with node $N$.
- Create a **visited set** and add node $N$.
- While queue is **not empty**:
    - pop a node $N'$ from the priority queue.
    - if $N'$ is the **target node** return.
    - Enqueue all **unvisited adjacent nodes ($N''$)** from $N'$ maintaining priority by $h(N'')$.
    - Add all **unvisited adjacent nodes** to the visited set.

## Pseudocode
```ts
const comparatorByHeuristic = (a:Node, b:Node) => {
    return h(a) < h(b); // Min-heap: lower h(n)
}

function greedyBestFirstSearch(node: Node, target: Node) {
    const priorityQueue = new minPriorityQueue(comparatorByHeuristic);
    priorityQueue.add(node)

    const visited = new Set<Node>();
    visited.add(node);

    while ( !priorityQueue.isEmpty() ) {
        const currentNode = priorityQueue.extractMin();
        if (currentNode === target) return;

        for (neighbour of currentNode.adjacent()) {
            if (visited.has(neighbour)) continue;
            
            priorityQueue.insert(neighbour);
            visited.add(neighbour);
        }
    }
}
```
